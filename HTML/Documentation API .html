ProductDB RESTful API by Ruben and Kostas

The product API can be used to save products in a database. The API is accessible via port 8080 and works via HTTP. The port can be modified in the server.py source code.

Create a new row
Retrieve a table or row
Update a row
Delete or reset a table or row
Error handling

Functions

Create a new row
  • Parameter: /create
  • Function: Creates a new row in the database when a JSON objecttable is sent in the request body.
  • Requestmethod: HTTP DELETE
  • Dataformat required: JSON
  • Response: HTTP 201
Retrieve a table or row
  • Parameter: /retrieve-all
  • Function: Used to retrieve the whole database in JSON format.
  • Requestmethod: HTTP GET
  • Response: HTTP 200
  • Parameter: /retrieve/<rownumber>
  • Function: Used to retrieve a specific row of the database.
  • Requestmethod: HTTP GET
  • Response: HTTP 200
Update a row
  • Parameter: /update/<rownumber>
  • function: Updates a row when a JSON objectlist is sent in the request body
  • Dataformat required: JSON
  • Requestmethod: HTTP PUT
  • Response: HTTP 200
Delete or reset a table or row
  • Parameter: /delete/<rownumber>
  • Function: Removes the specified rownumber from the database.
  • Requestmethod: HTTP DELETE
  • Response: HTTP 200
  • Parameter: /reset
  • Function: Resets the whole database and will put one example product in the database.
    The example product in JSON format: ('Banana', 'Fruit', 'Amsterdam', '2014-10-05', '15')
  • Requestmethod: HTTP DELETE
  • Response: HTTP 200
  • Parameter: /reset-all
  • Function: Resets the whole database to an empty state.
  • Requestmethod: HTTP DELETE
  • Response: HTTP 200

Errorhandling

Error 404
When a link does not match one of the above parameters, a 404 will be raised.
Error 415
The server expects data in (valid) JSON format in the requestbody when accepting the ‘create’ or ‘update’ requests. If no valid JSON data is detected in the body error 415 is raised.
